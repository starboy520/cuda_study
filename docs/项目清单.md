# CUDA 实战项目清单

12 个递进项目 + 3 个作品集候选。每个项目建议包含：

- `README.md`：问题、方法、结果表
- 可复现构建（CMake/Makefile）
- 至少 1 张性能对比表或 Nsight 截图

---

## 阶段一：基础（Week 1–2）

### P01 · 向量加法与设备查询 `★ 必做`

**目标**：熟悉编译流程、内存拷贝、错误检查。

**验收**

- [ ] N = 1M~100M 可配置
- [ ] `CUDA_CHECK` 宏包装 API
- [ ] 打印 T4 全部关键 `cudaDeviceProp` 字段
- [ ] 对比 `cudaMemcpy` 同步 vs 异步耗时（Week 3 可补）

**扩展**：float / double / float4 向量加载对比。

---

### P02 · Naive 矩阵乘法 `★ 必做`

**目标**：建立 GEMM 性能基线。

**验收**

- [ ] 支持 M=N=K 可配置，默认 512/1024/2048
- [ ] 输出 GFLOPS = 2MNK / time
- [ ] 与 CPU 单线程结果对比（误差 < 1e-3）

**基线记录表（示例）**

| Size | Time (ms) | GFLOPS |
|------|-----------|--------|
| 1024³ | | |

---

### P03 · 矩阵转置三版对比 `★ 必做`

**目标**：理解合并访问与 shared memory。

**版本**

1. naive 索引转置
2. coalesced 读 + 写不合并
3. shared memory tile + padding 避 bank conflict

**验收**

- [ ] 4096×4096，测有效带宽 GB/s
- [ ] Nsight Compute 看 `memory_throughput` 差异

---

### P04 · 并行归约 `★ 必做`

**目标**：掌握 tree reduction、warp shuffle。

**版本**

1. 原子 global reduction（慢，作对比）
2. shared memory 树形归约
3. warp shuffle + 单 block 多 warp

**验收**

- [ ] 1M float sum，误差 < 1e-2
- [ ] 记录各版本耗时

---

## 阶段二：进阶（Week 3–4）

### P05 · Prefix Sum (Scan) `★ 必做`

**目标**：Work-efficient Blelloch scan。

**验收**

- [ ] inclusive / exclusive 两种
- [ ] 1M 元素，与 Thrust `inclusive_scan` 对比耗时

---

### P06 · Stream 流水线 `★ 必做`

**目标**：H2D、kernel、D2H 重叠。

**验收**

- [ ] 双缓冲或 2+ stream
- [ ] timeline 证明重叠（nsys screenshot）
- [ ] 端到端加速比 ≥ 15%（数据量要够大）

---

### P07 · cuBLAS GEMM 封装 `★ 必做`

**目标**：工业库调用与 leading dimension。

**验收**

- [ ] 行主序 vs 列主序说明清楚
- [ ] 1024³ FP32，记录 cuBLAS GFLOPS
- [ ] 与 P02 naive 算加速比

---

### P08 · CMake 统一工程 `推荐`

**目标**：工程化构建所有周项目。

**验收**

- [ ] `cmake -B build && cmake --build build`
- [ ] `ctest` 或自定义 `make test` 跑正确性

---

## 阶段三：优化（Week 5–6）

### P09 · Tiled GEMM + Roofline `★ 作品集候选`

**目标**：分块、double buffer 入门、性能方法论。

**版本递进**

1. naive (P02)
2. shared memory tile
3. 寄存器 blocking（可选）
4. WMMA FP16 tile（可选）

**验收**

- [ ] Roofline 图 + 各版 GFLOPS 表
- [ ] Nsight Compute：说明 bound 类型与证据
- [ ] README 写「为何仍慢于 cuBLAS」三点分析

**作品集权重**：⭐⭐⭐⭐⭐

---

### P10 · Benchmark 框架 `推荐`

**目标**：可复现性能测试。

**功能**

- warmup iterations
- 多次取 median / p99
- 输出 CSV/JSON
- `cudaEvent` 计时

**验收**

- [ ] 一行命令对比两个 kernel 实现
- [ ] 文档说明测试环境（GPU、驱动、CUDA 版本）

---

## 阶段四：应用（Week 7–8）

### P11 · 方向项目（三选一）`★ 作品集主打`

#### A · PyTorch 自定义 CUDA 算子

- 实现 fused bias+ReLU 或 Layernorm
- `torch.utils.cpp_extension` 编译
- 对比 `torch.nn` 原生速度

#### B · 稀疏 SpMV

- CSR 格式 SpMV kernel
- 与 cuSPARSE 对比

#### C · 图像处理 Pipeline

- 高斯模糊 + Sobel（shared memory 卷积）
- 对比 OpenCV CPU
- 可选：接 ONNX 小模型做「预处理+推理」

**验收**

- [ ] 端到端延迟分解表
- [ ] 与业务场景挂钩的一句话描述（简历用）

---

### P12 · 推理优化 Mini `★ 作品集候选（推理岗）`

**目标**：TensorRT 或 ONNX Runtime CUDA EP。

**流程**

1. PyTorch → ONNX
2. TensorRT build（FP16/INT8）
3. 测 latency / throughput

**T4 上**

- ResNet18/50 或小型 Transformer encoder
- 记录 batch=1/8/32 的 FPS

**作品集权重**：⭐⭐⭐⭐（推理岗）
