# Tesla T4 实战指南

你的环境：**Tesla T4 · 16GB GDDR6 · Compute Capability 7.5 · Driver 595 / CUDA 13.x**

本指南帮助你在 T4 上设定合理预期、避免常见硬件误区，并把实验结果写进作品集。

> 若关心 **T4 vs A100、哪些学习内容与卡型相关**，见 [GPU卡型专项学习指南.md](GPU卡型专项学习指南.md)。

---

## 一、T4 硬件速览

| 规格 | 数值 | 学习意义 |
|------|------|----------|
| 架构 | Turing (2018) | 支持 Tensor Core FP16/INT8，无原生 FP8 |
| SM 数量 | 40 | 理解 grid 规模与 occupancy |
| CUDA Cores | 2560 | FP32 主力 |
| Tensor Cores | 320 (2nd gen) | Week 5 WMMA 实验 |
| 显存 | 16 GB GDDR6 | 足够大模型推理实验（中小模型） |
| 显存带宽 | ~300 GB/s | Roofline 带宽臂 |
| FP32 峰值 | ~8.1 TFLOPS | Roofline 算力臂 |
| FP16 Tensor | ~65 TFLOPS | 推理/混合精度方向 |
| TDP | 70W | 边缘推理常见卡，注重能效 |
| MIG | 不支持 | 与 A100/H100 不同，整卡使用 |

**与 A100/H100 的差异（面试常问）**

- T4：消费/推理向，无 NVLink（单卡学习为主）
- A100：训练向，HBM 高带宽、MIG、NVLink
- 编程模型相同，**优化思维可迁移**，峰值数字不可直接对比

## 二、编译与架构标志

T4 对应 `sm_75`：

```bash
# 单次编译
nvcc -O3 -arch=sm_75 -o app app.cu

# 多架构（若代码需在多种 GPU 跑）
nvcc -O3 -gencode arch=compute_75,code=sm_75 -o app app.cu
```

**CMake 示例**

```cmake
set(CMAKE_CUDA_ARCHITECTURES 75)
```

查询设备确认：

```bash
nvidia-smi --query-gpu=name,compute_cap,memory.total --format=csv
# 应显示 7.5, 16384 MiB
```

## 三、T4 上的性能基准参考（FP32）

以下为 **学习实验合理区间**（非官方峰值，受实现影响大）：

| Kernel / 操作 | 朴素实现 | 优化后目标 | cuBLAS/库 |
|---------------|----------|------------|-----------|
| 向量加 1M | 内存带宽 ~50% | ~70% 峰值带宽 | N/A |
| 矩阵转置 4096² | ~30 GB/s | ~200+ GB/s | N/A |
| GEMM 1024³ FP32 | ~50 GFLOPS | 500–1500 GFLOPS | ~3000+ GFLOPS |
| Reduction 1M | — | 与带宽相关 | Thrust 很快 |

**重要心态**

- 手写 GEMM **追不上 cuBLAS 是正常的**，作品集写清「达到 cuBLAS 的 X%」即可
- 每次优化记录 **绝对值 + 相对基线**，比空谈「优化了很多」更有说服力

## 四、Roofline 在 T4 上的用法

**FP32 粗略参数**

- 峰值算力 \(P\) ≈ 8.1 TFLOPS = 8.1×10¹² FLOP/s
- 峰值带宽 \(B\) ≈ 300 GB/s = 300×10⁹ B/s
- 拐点算术强度 \(I^* = P/B\) ≈ **27 FLOP/Byte**

**判断**

- GEMM 大矩阵：通常 **compute-bound**（强度远高于 27）
- 向量加、转置、element-wise：通常 **memory-bound**
- 小矩阵 GEMM：可能 **latency-bound**（launch、占用率低）

Week 5 用 Python 画 roofline，把自己的 kernel 点标上去。

## 五、显存预算习惯

16GB 看似充裕，养成预算表（推理岗必备）：

| 组件 | 估算 |
|------|------|
| 模型权重 FP32 | 参数量 × 4 B |
| 模型权重 FP16 | 参数量 × 2 B |
| 激活值 | batch × 特征图大小 × 精度 |
| 中间 buffer | 算子融合可减少 |
| cuBLAS workspace | 调用时查询 |

**示例**：ResNet50 FP32 权重约 100MB，推理 batch=32 远小于 16GB；可用来练 **端到端 pipeline**，而非只看 kernel。

## 七、功耗与稳定性

```bash
# 持续跑 benchmark 时观察
watch -n 1 nvidia-smi

# 性能模式（若被 power capping）
sudo nvidia-smi -pm 1
sudo nvidia-smi -pl 70   # T4 最大约 70W
```

- 数据中心 T4 通常散热良好；长时间训练注意温度墙
- 学习阶段 **固定 GPU 时钟** 可让 benchmark 更可复现（高级，Week 6 可选）

## 八、常见问题

### Q: T4 适合练训练吗？

适合 **小模型 / 单卡训练原理**；大模型分布式训练思维用 NCCL 文档 + 云 GPU 补充，简历可写「T4 单卡验证，理解多卡扩展」。

### Q: 必须用 Tensor Core 吗？

推理向岗位 **建议会 WMMA/FP16**；至少跑通一个样例并解释与 FP32 路径差异。

### Q: CUDA 13.x 与网上老教程不一致？

以你本机 `/usr/local/cuda` 文档为准；旧教程的 `sm_60` 等需改成 `sm_75`。

### Q: 没有第二块 GPU 怎么学多卡？

学 **NCCL API 概念** + 读源码/文档；有条件用云 A100 几小时做 AllReduce 实验即可，非必须。

