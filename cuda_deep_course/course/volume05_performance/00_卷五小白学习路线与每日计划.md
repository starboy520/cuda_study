# 00 卷五小白学习路线与每日计划

这份文档是卷五的入口。

你明天开始学性能优化时，不建议直接跳到 Nsight Compute 或 Roofline。性能优化最容易乱，是因为
你同时面对：

```text
时间
带宽
GFLOPS
occupancy
stall
Nsight Systems
Nsight Compute
Compute Sanitizer
Roofline
```

这些词不是平级的。正确学习顺序是：

```text
先会可靠测量
再会计算指标
再会判断瓶颈
再会使用工具验证
最后才改代码优化
```

## 1. 卷五到底要学会什么

学完卷五，你要具备一个能力：

> 看到一个 CUDA kernel 慢，不靠猜，而是能用数据找原因，再提出一个可验证的优化假设。

完整流程：

```text
1. 确认正确性。
2. 可靠计时。
3. 判断端到端瓶颈还是 kernel 瓶颈。
4. 计算 GB/s、GFLOPS、AI。
5. 用 Roofline 判断大方向。
6. 用 Nsight Systems 看系统时间线。
7. 用 Nsight Compute 看 kernel 内部。
8. 提出一个优化假设。
9. 修改一个因素。
10. 复测并写报告。
```

这就是性能工程。

## 2. 不要一开始就做的事

先不要：

```text
一上来改 block size。
一上来加 shared memory。
一上来追 occupancy 100%。
一上来读一堆 Nsight 指标。
一上来想“怎么让它更快”。
```

先问：

```text
它真的慢吗？
慢在哪里？
瓶颈是端到端、传输、launch、kernel，还是某一个 kernel 内部？
```

优化之前先定位。

## 6. 小白最容易混的概念

### Time 和 Throughput

```text
Time:
  一次任务花多久。

Throughput:
  单位时间完成多少工作。
```

延迟服务看 time / P99。

批处理和训练看 throughput。

### GB/s 和 GFLOPS

```text
GB/s:
  搬数据速度。

GFLOPS:
  浮点计算速度。
```

vector add 看 GB/s。

GEMM 看 GFLOPS。

### Occupancy 和 Utilization

```text
Occupancy:
  理论上 SM 上驻留了多少 warp。

Utilization:
  执行单元实际忙不忙。
```

高 occupancy 不保证高 utilization。

### Nsight Systems 和 Nsight Compute

```text
Nsight Systems:
  看系统时间线，回答“时间去哪了？”

Nsight Compute:
  看单个 kernel 内部，回答“这个 kernel 为什么慢？”
```
