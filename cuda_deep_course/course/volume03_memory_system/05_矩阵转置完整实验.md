# 05 矩阵转置完整实验

## 1. 问题

输入：

```text
height 行 x width 列
```

输出：

```text
width 行 x height 列
```

关系：

```text
output[col][row] = input[row][col]
```

行主序地址：

```cpp
input[row * width + col]
output[col * height + row]
```

## 2. Naive 版本

```cpp
output[col * height + row] =
    input[row * width + col];
```

`threadIdx.x=col` 时，warp 连续读取 input；但写 output 时相邻 lane 跨越
`height`，写地址不连续。

## 3. Shared 版本的两层转置

完整坐标由两层组成：

```text
block 坐标 + block 内坐标
```

因此转置要同时完成：

```text
输入 block(rowBlock,colBlock)
-> 输出 block(colBlock,rowBlock)

tile[localRow][localCol]
-> tile[localCol][localRow]
```

## 4. 为什么 block 是 32 x 8，tile 是 32 x 32

```cpp
dim3 block(32, 8);
```

每个 warp 固定 `threadIdx.y`，`threadIdx.x=0..31`，适合连续访问一行。

每个 thread 循环处理四行：

```cpp
for (int offset = 0; offset < 32; offset += 8) {
  // y + 0, 8, 16, 24
}
```

总共覆盖 `32 x 32` tile，但只使用 256 个 thread，而不是 1024 个。

## 5. 读取

```cpp
tile[threadIdx.y + offset][threadIdx.x] =
    input[inputRow * width + inputCol];
```

warp 的 `inputCol` 连续，因此 global load 合并。

## 6. 写出

```cpp
output[outputRow * height + outputCol] =
    tile[threadIdx.x][threadIdx.y + offset];
```

Block 坐标交换：

```text
outputRow base = blockIdx.x * 32
outputCol base = blockIdx.y * 32
```

Tile 下标交换完成内部转置。

## 7. 非方阵边界

读取检查：

```cpp
inputRow < height && inputCol < width
```

写出检查：

```cpp
outputRow < width && outputCol < height
```

输入与输出宽高已经交换，不能复用同一组边界含义。

## 8. Padding

```cpp
__shared__ float tile[32][33];
```

第 33 列只用于改变行跨度，减少按列读取时的 bank conflict。

## 9. 运行

```bash
make -C labs/03_memory_system/transpose clean all
./labs/03_memory_system/transpose/transpose
./labs/03_memory_system/transpose/transpose 1024 768
./labs/03_memory_system/transpose/transpose 31 33
```

默认 `1003 x 769` 特意同时覆盖非方阵和非 tile 整除情况。

## 10. Profiling

```bash
ncu --set full \
  --kernel-name regex:transposeNaive \
  ./labs/03_memory_system/transpose/transpose 4096 4096

ncu --set full \
  --kernel-name regex:transposeShared \
  ./labs/03_memory_system/transpose/transpose 4096 4096
```

观察：

- DRAM throughput。
- global load/store efficiency 或相关 sector 指标。
- shared bank conflict。
- source counters。

## 11. 故障注入

1. 删除 `__syncthreads()`。
2. 将输出边界错误写成输入边界。
3. 将 `[32][33]` 改为 `[32][32]`。
4. 将 shared 读取写成 `tile[y][x]`，观察结果不再转置。

正确性故障用：

```bash
compute-sanitizer --tool memcheck ./labs/03_memory_system/transpose/transpose
compute-sanitizer --tool racecheck ./labs/03_memory_system/transpose/transpose
```
