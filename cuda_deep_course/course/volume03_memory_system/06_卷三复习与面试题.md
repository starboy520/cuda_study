# 06 卷三复习与面试题

## 核心问题

1. Register 与 local memory 有什么区别？
2. Shared memory 的共享范围和生命周期是什么？
3. Global memory 为什么可以同时“高延迟、高带宽”？
4. 什么是合并访问？分析单位为什么是 warp？
5. 对齐如何影响事务数量？
6. AoS 与 SoA 各适合什么场景？
7. Block 形状和 tile 形状为何不同？
8. `__syncthreads()` 解决什么问题？
9. 什么是 bank conflict？
10. 为什么 transpose 常声明 `[32][33]`？
11. Pinned memory 的收益和代价是什么？
12. Unified Memory 是否意味着没有数据迁移？

## 手工题

### 事务覆盖

32 个 lane 各读取连续一个 `float`，起点地址为 0。列出覆盖的 32-byte 区间。

### Bank

对 `float tile[32][32]`，固定列读取时计算：

```text
bank = (row * 32 + col) % 32
```

再对 `[32][33]` 重算。

### 非方阵

输入 `7 行 x 10 列`：

- 输出尺寸是什么？
- input/output 一维下标分别乘什么？
- shared transpose 的读写边界是什么？

## 面试回答结构

回答“如何优化 global memory”时不要只说 coalescing。完整结构：

1. 减少传输和访问次数。
2. 让 warp 地址合并且对齐。
3. 改善数据布局。
4. 使用 register/shared/cache 复用。
5. 合并 Host-Device 传输并使用 pinned memory。
6. 用 profiler 验证真实瓶颈。
