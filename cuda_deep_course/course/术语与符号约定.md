# 术语与符号约定

全书统一使用本页约定。遇到特殊 API 或库采用不同约定时，章节必须单独说明。

## 1. Host 与 Device

```text
Host   = CPU 及其可直接访问的内存
Device = GPU 及其可直接访问的资源
```

常见变量命名：

```cpp
float* h_data;  // host data
float* d_data;  // device data
```

## 2. 线程坐标

二维问题默认采用最常见的直觉：

```text
x = 横向 = column = col = 列
y = 纵向 = row    = row = 行
```

所以：

```cpp
const int col = blockIdx.x * blockDim.x + threadIdx.x;
const int row = blockIdx.y * blockDim.y + threadIdx.y;
```

其中：

```text
threadIdx.x = block 内的列
threadIdx.y = block 内的行
blockIdx.x  = block 的列
blockIdx.y  = block 的行
```

## 3. 矩阵尺寸

矩阵写作：

```text
height 行 x width 列
```

例如：

```text
height = 3
width  = 5

矩阵形状 = 3 行 x 5 列
```

不要把“`3 x 5`”理解成 `width x height`。正文第一次出现尺寸时会同时写出
“行”和“列”。

## 4. 行主序

除非章节明确讨论列主序，否则矩阵默认使用行主序：

```cpp
matrix[row * width + col]
```

因为每一行有 `width` 个元素：

```text
跳过前面的 row 行：row * width
再向右移动 col 个元素：+ col
```

## 5. 矩阵转置

输入：

```text
height 行 x width 列
```

输出：

```text
width 行 x height 列
```

坐标关系：

```text
input[row][col] -> output[col][row]
```

一维下标：

```cpp
input[row * width + col]
output[col * height + row]
```

输出下标乘 `height`，因为输入的高度变成了输出的宽度。

## 6. Block 与 Tile

这两个概念不能混用：

```text
block 形状 = 线程如何排列
tile 形状  = 一个 block 负责的数据区域
```

例如：

```text
block = 32 x 8 个线程
tile  = 32 x 32 个矩阵元素
```

每个线程循环处理 4 个元素时，这种组合完全合理。

## 7. 二进制与十进制单位

内存容量优先使用二进制单位：

```text
1 KiB = 1024 bytes
1 MiB = 1024 KiB
1 GiB = 1024 MiB
```

带宽通常按厂商规格使用十进制单位：

```text
1 GB/s = 1,000,000,000 bytes/s
```

实验中必须说明采用哪种单位。

## 8. 时间与性能

```text
latency    = 完成一次工作的时间
throughput = 单位时间完成的工作量
bandwidth  = 单位时间传输的数据量
```

常用指标：

```text
ms      = 毫秒
us      = 微秒
GB/s    = 十进制每秒十亿字节
GFLOPS  = 每秒十亿次浮点运算
```

峰值规格、公式估算和实际测量必须分别标注。

## 9. 代码版本命名

复杂实验按以下方向递进：

```text
cpu_reference
naive
correct
tiled
memory_optimized
warp_or_register_optimized
architecture_specific
```

版本名不代表一定更快。优化效果必须通过测量确认。
