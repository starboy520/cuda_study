# 卷十：工程化、作品集与面试

> 前九卷教你把 CUDA **写对、写快、讲清原理**。卷十回答最后一个问题：**怎么把这些能力
> 变成"别人信得过的工程"和"面试拿得出手的表达"**。从"能写 kernel"到"能交付一个可维护、
> 可复现、有测试有基线的项目"，再到"能在白板上手写 + 讲清优化"，这一卷是临门一脚。

## 学习目标

- 用 CMake 管理可复现、可跨架构的 CUDA 构建。
- 用 RAII + 错误包装让资源生命周期安全、错误无处遁形。
- 建立 CPU reference + 随机/边界测试 + 误差容忍的正确性体系。
- 建立性能基线与回归测试，让"变快了"和"变慢了"都有数据。
- 理解 Compute Capability / PTX JIT 的部署兼容性，避免"换台机器跑不了"。
- 用一份 code review 清单审自己和别人的 kernel。
- 建一个常见故障案例库，遇到症状能反查原因。
- 系统准备面试：概念题、性能分析题、手写 kernel 题、系统设计题。
- 用三个作品集项目把全部能力打包成可展示的成果。

## 为什么需要这一卷（一张图）

```text
会写 kernel  ─┐
能调快     ─┼─► 但如果：构建不可复现、没测试、没基线、讲不清
能讲原理   ─┘     → 面试官无法判断你的真实水平，团队不敢用你的代码

卷十补上：可复现构建 + 测试基线 + 故障库 + 面试表达 + 作品集
        = 把"我会"变成"别人信"
```
