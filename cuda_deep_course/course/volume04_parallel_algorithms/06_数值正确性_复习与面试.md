# 06 数值正确性、复习与面试

## 1. 浮点加法不满足结合律

数学上：

```text
(a + b) + c = a + (b + c)
```

浮点中舍入可能导致两边不同。并行 reduction 改变求和树，所以结果可能与
串行顺序不同。

## 2. 验证不能只用 `==`

常见混合判断：

```text
abs(actual - expected)
<= absoluteTolerance + relativeTolerance * abs(expected)
```

容差取决于：

- 数据规模。
- 数值范围。
- 运算次数。
- 算法稳定性。
- 数据类型。

## 3. 改善精度

- 使用 double 累加。
- Pairwise/tree summation。
- Kahan 或 compensated summation。
- 缩放数据。
- 避免灾难性消减。

精度通常有性能和资源成本，应由需求决定。

## 4. 非确定性

Atomic 浮点归约中，不同运行的更新顺序可能不同，因此末位结果变化。

若业务要求 bitwise reproducibility，需要：

- 固定归约结构。
- 避免不确定顺序 atomic。
- 控制编译和硬件差异。
- 接受额外成本。

## 7. 故障注入

1. 删除 shared reduction 循环中的 barrier。
2. 将 `start + blockDim.x < count` 删除。
3. 在 partial warp 中错误使用 full mask。
4. 使用严格 `float == doubleReference` 判断。

分别解释错误属于：

```text
race
越界
warp participation contract
数值验证错误
```
